Space Value Check

 

Overview

The purpose of this rule is to check that values in the model are the same as the given ones. The rules is made for check the testing model (1800 F model) and it shouldn't be used if the required data is not available.

Configuring the Constraint

The rule have the following parameters:

Required Values

A table for the values. The key for the spaces is the space number. Columns of the table are

  • Space Number

  • Space Name

  • GSA STAR Type

  • GSA STAR Category

  • ANSI/BOMA Space Category

  • Occupant Organization Name

  • Occupant Organization Code

  • Occupant Organization Abbreviation

  • Occupant Sub-Organization Name

  • Occupant Billing ID

  • Security Zone

  • Preservation Zone

  • Privacy Zone

Values can be read from Excel spreadsheet.

GSA Space Categories

Analyzing the Results

The rule creates issues for missing spaces based on space numbers. The model should include all spaces listed in the table. The rule creates issues for extra spaces (spaces not listed in the table). The model shouldn't include any extra spaces. Note that space groups are not taken into account in this rule at all.

The rule creates issues for each wrong values (value which differ from the value in the table), Spaces with the same name are collected to same sub category.

If the model doesn't have required property sets or properties, issues are created. This helps you to identify correct property set and property names (used in the model),

Reported Data

The rule does not have a report.

Constraint Tools

The rule does not have any tools.
