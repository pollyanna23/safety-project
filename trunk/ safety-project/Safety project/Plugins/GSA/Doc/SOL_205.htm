Area Comparison Rule

 

Overview

The rule checks that given Net Area, Usable Area and Rentable Area. The rule is supposed to use only in testing purposes (it is made for F 1800 test case).

Configuring the Constraint

The rule have the following parameters:

GSA Space Area

Name of the property set in the model. In some cases the property set names are not correct in the model. You can specify the used current property set name by using this parameter.

GSA BIM Area

Name of the property in the model. In some cases the property names are not correct in the model. You can specify the used current property name by using this parameter.

GSA Usable Area

Name of the property in the model. In some cases the property names are not correct in the model. You can specify the used current property name by using this parameter.

GSA Rentable Area

Name of the property in the model. In some cases the property names are not correct in the model. You can specify the used current property name by using this parameter.

 

Analyzing the Results

The rule creates issues when given areas differs from value calculated by SMC more than 0.5%. Every space creates an own issue. Issues are organized according the difference size (%).

The rule creates issues also when required properties don't exists or if its value is null.

Reported Data

The rule does not have a report.

Constraint Tools

The rule does not have any tools.
