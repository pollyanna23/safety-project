Walls Between Public, Restricted, and Secure Spaces

Overview

This rule checks that walls around secure areas are secure by types.

Running this rule with the default values does not give useful results; this rule has to be configured before use.

Configuring the Rule

The rule has the following parameters:

Secure Wall Types

A list of secure wall types.

Acceptable Opening Size

Acceptable area of an empty opening in a wall between secure and other areas.

Acceptable Gap

An acceptable gap between walls around a space.

Tolerance

A tolerance for searching walls around spaces. Walls, which are nearer than the tolerance, are taken into account.

Analyzing the Results

If there is a non-secure wall between secure and some other (public, restricted) area, an issue is created and the wall is attached to it.

If there are too large openings in walls between secure and some other (public, restricted) area, an issue is created. If a secure wall does not touch a slab above or below, an issue is also created.

All spaces, whose secure level is not defined, create also on issue.

Reported Data

The rule doesn't have a report.

Rule Tools

The tools panel of the rule includes buttons to define space secure level (public, restricted, or secure). You can either set secure level for all spaces with the same name and type, or an individual space. You can also highlight spaces with a specific secure level in the 3D View. Note that secure level is gotten from GSA Space Data.

Tools panel allows you also to set walls to be secure. You can either set type of the selected walls to be among the secure wall types, or set an individual wall to be secure. You can also highlight all secure walls in the 3D view.
